# 智能客服功能头脑风暴

**日期**: 2026-02-28
**状态**: 待规划

## What We're Building

在工单表单页右下角添加智能客服入口，帮助**内部售后人员**在创建工单前自助解决问题，减少不必要的工单创建。

**核心价值**：利用历史已完成工单的知识，通过 AI 引导用户找到解决方案，实现"能自助解决的不创建工单"。

## Why This Approach

### 为什么只在工单表单页显示

- **场景聚焦**：用户在创建工单时最需要"先尝试自助解决"
- **减少干扰**：不需要在所有页面都显示，保持界面简洁
- **自然流程**：自助解决失败 → 顺滑过渡到创建工单

### 为什么选择多轮引导式对话

- **问题质量**：单次问答容易描述不清，导致推荐不准确
- **用户体验**：AI 主动引导比让用户"一次说清楚"更友好
- **匹配现有能力**：复用现有的向量搜索和智能推荐逻辑

### 为什么需要用户确认问题

- **准确性保障**：确保搜索的是用户真正想问的问题
- **可控性**：用户可以修正 AI 的理解，避免推荐偏题

## Key Decisions

| 决策项 | 决定 | 理由 |
|--------|------|------|
| 目标用户 | 内部售后人员 | 利用完整历史工单数据，减少内部重复工单 |
| 触发位置 | 仅工单表单页 | 场景聚焦，减少干扰 |
| 交互方式 | 多轮引导对话 | 确保问题描述充分，推荐更准确 |
| 判断机制 | AI 自动判断 | 无需用户手动操作，体验更流畅 |
| 问题确认 | AI 总结 + 用户确认 | 确保搜索意图准确 |
| 数据来源 | 向量数据库（已完成工单） | 已有数据，无需额外维护知识库 |
| 解决后处理 | 记录自助解决事件 | 可统计解决率，评估效果 |
| 对话历史 | 保留 1 天 | 避免数据膨胀，MongoDB 定时清理 |
| 推荐展示 | 文字推荐 + 相似工单卡片 | 兼顾便捷性和可追溯性 |
| 解决统计 | 仪表盘展示 | 方便查看智能客服效果 |
| 预设问题 | 不需要 | 保持简洁，第一版专注核心流程 |

## 技术方案概要

### 后端

- 新增 `/api/chat` 端点，处理对话请求
- 新增 `ChatSession` 和 `ChatMessage` 模型，存储对话历史（1 天过期）
- 新增 `SelfServiceRecord` 模型，记录自助解决事件
- 定时任务清理过期的对话历史数据
- 复用 `ai_service.py`：
  - `find_similar_tickets()` - 相似工单搜索
  - `generate_handling_recommendation()` - 智能推荐生成
- 新增对话状态机逻辑：
  - `DESCRIBING` → `CONFIRMING` → `SEARCHING` → `RECOMMENDED` → `RESOLVED/CANCELLED`

### 前端

- 新增 `SmartAssistant` 悬浮聊天组件
- 仅在 `TicketForm` 页面渲染
- UI 风格复用现有科技风深色主题
- 状态管理：对话消息列表、当前阶段、加载状态
- 展示：AI 推荐文字 + 相似工单卡片列表

### 仪表盘

- 新增自助解决统计卡片
- 展示：自助解决数量、解决率
